# git log/reflog: Verificando o histórico do git Para ver o histórico de commits do **Git**, digite: ``` $ git log ``` - **$** indica que você deve usar o **usuário comum** para fazer essa operação. E o retorno será todos os **commits** feitos até o momento. ![mostrando todos os commits](img/p0005-0.png) Pressione [ENTER] quantas vezes sejam necessárias para ver os outros commits (isso ocorre quando o número de commits é maior do que o espaço ocupado pelo terminal) e ao final, quando aparecer a palavra **(END)**, pressione a letra **q** para sair. ![final da lista do histórico de commits](img/p0005-1.png) Caso queira ver esse histórico numa versão mais resumida, use a flag apresentada abaixo: ``` $ git log --oneline ``` ![usando a flag oneline](img/p0005-2.png) Caso queira ver apenas os 4 últimos **commits**, ``` $ git log -4 ``` ![4 últimos commits](img/p0005-3.png) Você pode substituir o número 4 por outro número inteiro, conforme a necessidade. Também é possível fazer uma combinação desses comandos. Exemplo: Usando o git log em combinação com a flag oneline. ![usando o git log em combinação com a flag oneline](img/p0005-4.png) Para ver o histórico de tudo que foi feito (não apenas commits), ``` $ git reflog ``` ![usando o reflog](img/p0005-5.png) Esse comando é muito útil para usar com o **reset** quando você precisa recuperar algo que foi perdido por acidente, seja depois de um **merge** ou próprio **reset** problemático, algo que quebrou ou foi excluído e até mesmo recuperar um **branch** excluído. Ele é como uma máquina do tempo mais potente do que o **log**. ``` $ git reset HEAD@{número_do_índice} ``` Entretanto, o **reflog** tem 90 dias de prazo de validade padrão.
